# 引言
在软件系统中，某些类型由于自身的逻辑，它具有两个或多个维度的变化，那么如何应对这种“多维度的变化”？
如何利用面向对象的技术来使得该类型能够轻松的沿着多个方向进行变化，而又不引入额外的复杂度？这就要使用桥接模式。

![桥接模式1.png](桥接模式1.png)

如果不用桥接模式,那么就变成了

![桥接模式2.png](桥接模式2.png)

看到没总共就生成了`2*2*2=8`种产品.如果其中一个`具体维度`又增加一种变化,那么就变成`2*3*2=12`,也就是说一些要增加4个具体产品类.
